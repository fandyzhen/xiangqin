# 大章丘脱单资格赛 H5 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a mobile-first WeChat H5 lead-capture game for Zhangqiu single men, following `../大章丘脱单资格赛-开发Spec-v1.md`.

**Architecture:** Next.js App Router hosts the H5 flow and two API routes. Pure domain modules own quiz selection/scoring, ideal-type prompt assembly, fallback portrait generation, match-card synthesis, and submission validation so they can be tested without browser state.

**Tech Stack:** Next.js, React, TypeScript, CSS, Vitest, html2canvas, qrcode, nodemailer.

---

## File Structure

- `app/page.tsx`: single-page H5 state machine, cover, quiz, result, ideal type, share card, match animation, lead form, success state.
- `app/layout.tsx`: page metadata and mobile viewport.
- `app/globals.css`: mobile-first visual system and interaction states.
- `app/api/ideal-image/route.ts`: returns prompt, seed, provider, and a generated/fallback image URL.
- `app/api/submit/route.ts`: validates submission, writes local JSONL backup, and sends email when SMTP env is configured.
- `lib/types.ts`: shared TypeScript contracts.
- `lib/questions.ts`: Appendix A question bank and result copy.
- `lib/quiz.ts`: fixed-structure quiz draw, scoring, eligibility, analysis.
- `lib/ideal.ts`: ideal-type options, prompt assembly, seed helpers, fallback portrait data URL.
- `lib/match.ts`: Zhangqiu residence pool and matched-girl synthesis.
- `lib/submission.ts`: form validation and email body formatting.
- `tests/domain.test.ts`: behavior tests for the core domain.
